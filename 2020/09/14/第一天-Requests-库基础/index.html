第一天:Requests 库基础

章鱼哥的个人空间
2020-09-14 (Updated: 2020-09-14)
Python
Crawler, Requests

第一天：

废话不多说，上列子：

理解Response的编码：

  • r.encoding: 从网页的 charset 中猜测编码形式，如果header中不存在charset，则设定默认编码为ISO-8859-1(不能显示中文)
  • r.apparent_encoding: 分析网页，获得编码形式

结论：r.apparent_encoding 比 r.encoding 更准确

爬取网页的流程：

  • r.status_code 来检查是否连接成功
    • 成功的话，用 r.text r.encoding r.apparent_encoding r.content 来解析网页内容
    • 失败的话：代表有错误

Requests 库基础：

  • r = re.get(url, params, **kwargs)

  • url = 网址

  • params = url中额外的参数

  • **kwargs = 12个控制访问的参数

  • r 是response对象，包含从服务器获得的所有信息

    Response 类的方法：

  • r.status_code # http请求的返回状态，200表连接成功，404是失败

  • r.text # 获得网络页面的字符串形式

  • r.encoding # 从http header中猜测出来的编码方式

  • r.apparent_encoding # 从内容分析出来的编码方式

  • r.content # 从二进制形式还原文件
